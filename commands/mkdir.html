Using the mkdir command

Description

Creates empty directories

Usage

mkdir [options] [directory name]

Examples

Creates a new directory named “rock” under the present working directory.

mkdir rock

Creates a new directory named rock under the music directory (relative path).

mkdir Music/rock

Creates two new directories called “rock” and “jazz”.

mkdir blues jazz

The “\” character is a escape character used for escaping a space.

mkdir /home/john/Music/rap\ music

You can surround the argument in quotation marks to avoid using the escape character (\)

mkdir “/home/john/music/Latin Music”

Let's see more complex examples:

Bob wants to create 3 directories in the Documents directory: math english linux, however, his present working directory is /usr/share How can he create the 3 directories without changing his present working directory?

Option 1) mkdir /home/$USER/Documents/math /home/$USER/Documents/english /home/$USER/Documents/linux
Option 2) mkdir ~/Documents/math ~/Documents/english ~/Documents/linux
Option 3) mkdir $HOME/Documents/math $HOME/Documents/english $HOME/Documents/linux
Option 4) mkdir ~/Documents/{math,english,linux}

Bob wants to make 2 directories: history biology, however, he also wants to make 2 more directories inside these 2 directories. How can he achieve this?

Option 1) mkdir -p Documents/history/fall Documents/history/spring Documents/biology/fall Documents/biology/spring
Option 2) mkdir -p Documents/history/{fall,spring} Documents/biology/{fall,spring}
Option 3) mkdir -p Documents/{history,biology}/{fall,spring}
